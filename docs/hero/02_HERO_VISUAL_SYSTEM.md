# HERO BIBLE

## PART 02 — Visual Direction & Layout System

Version: 1.0
Status: Approved
Language: English
Audience: AI Frontend Engineer (Gemini), UI/UX Designer, Frontend Developer

---

# Introduction

This document defines how the Hero should look.

Not how it should be coded.

Not how it should be animated.

But how visitors should visually experience it.

Every layout decision must support storytelling.

Every visual decision must support trust.

The Hero should feel carefully designed rather than heavily designed.

---

# Visual Philosophy

The Hero should immediately communicate:

Premium.

Professional.

Human.

Elegant.

Authentic.

Modern.

Calm.

Purposeful.

The Hero should never feel like a startup template.

It should never resemble a SaaS landing page.

It should never resemble a government website.

Instead, it should feel like a premium digital product built for a meaningful organization.

# Overall Composition

The Hero should immediately feel balanced.

Not symmetrical.

Balanced.

Visual weight should naturally lean toward the right side through the Browser Mockup and documentation collage.

The left side should remain calm and readable.

The right side should feel alive without becoming chaotic.

---

# Layout Ratio

Desktop

Approximately

45%

Content

↓

55%

Visual Experience

Never use a perfect 50/50 split.

The visual side should feel slightly more dominant.

Because it silently communicates credibility.

# Reading Flow

The eye should naturally travel like this.

Headline

↓

Description

↓

Primary CTA

↓

Browser Mockup

↓

Documentation Photos

↓

Floating Evidence

↓

Scroll

Never break this reading sequence.

---

# Hero Height

Desktop

85vh

Never use full-screen 100vh unless there is a specific reason.

Visitors should subconsciously notice that another section exists below.

This creates curiosity.

Scrolling should feel natural.

Not forced.

---

# Container Width

Use a maximum content width around:

1280px

Never stretch content edge-to-edge.

Premium interfaces breathe.

Whitespace is an intentional design element.

---

# Whitespace Philosophy

Whitespace is not empty.

Whitespace guides attention.

Whitespace creates confidence.

Whitespace improves readability.

Whitespace increases perceived quality.

Whenever unsure,

choose more whitespace.

Never compress content simply to fill space.

# Typography Hierarchy

Headline

Very large.

Bold.

Confident.

Short.

Maximum three lines.

Never become a paragraph.

---

Supporting Copy

Readable.

Human.

Comfortable.

Around 60–75 characters per line.

Never exceed comfortable reading width.

---

Buttons

Simple.

Large enough to touch comfortably.

Rounded.

Minimal.

---

# Headline Rules

The headline should communicate emotion before information.

Avoid technical language.

Avoid buzzwords.

Avoid corporate jargon.

Prioritize meaningful language.

Examples of emotional direction:

Building Communities.

Creating Impact.

Ideas Into Action.

Innovation With Purpose.

Empowering Local Potential.

The wording should feel timeless.

---

# Browser Mockup

The Browser Mockup is the visual centerpiece.

It is not decoration.

It represents the digital face of UTBEX.

Requirements:

Modern browser chrome.

Rounded corners.

Premium shadow.

Glass-like appearance.

Subtle border.

Authentic screenshot.

High quality.

The browser should slightly overlap the background.

It should appear to float.

---

Surround the Browser Mockup with authentic documentation.

Possible categories:

Awards.

Community Training.

Government Collaboration.

Workshop.

Branding Activities.

Mentoring.

Innovation.

Entrepreneurship.

Each image should support credibility.

Never use random photos.

Never use repeated photos.

Community Empowerment

Government Collaboration

Innovation Program

Workshop

Mentoring

# Background System

The background should feel invisible.

Visitors should notice it only when removed.

Layers:

Layer 01

Pure white foundation.

---

Layer 02

Very subtle mesh gradient.

---

Layer 03

Large radial glow.

Mostly behind the Browser Mockup.

---

Layer 04

Soft noise texture.

Very low opacity.

---

Layer 05

Blurred abstract shapes.

Barely visible.

---

Layer 06

Very subtle grid pattern.

Optional.

Only if it supports the composition.

---

# Color Strategy

Primary

White.

Off White.

Light Gray.

Charcoal.

Brand Red.

The Brand Red should appear only where attention is required.

Never overuse it.

Color should create rhythm.

Not decoration.

---

# Shape Language

Prefer:

Large radius.

Rounded rectangles.

Soft edges.

Smooth transitions.

Avoid:

Sharp corners.

Aggressive angles.

Heavy borders.

---

# Icon Language

Minimal.

Outline style.

Consistent stroke width.

Never mix icon styles.

Icons should support.

Never dominate.

---

# Visual Balance

The Hero should always maintain asymmetrical balance.

The left side carries informational weight.

The right side carries emotional weight.

Neither side should overpower the other.

---

# Scroll Hint

Never use a giant arrow.

Never use bouncing indicators.

Instead,

allow a small portion of the next section to become visible.

Visitors should naturally discover that more content exists.

Curiosity is stronger than instruction.

---
