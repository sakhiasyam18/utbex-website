---

# Objective

The desktop UI has been approved.

The purpose of this review is NOT to redesign the website.

The purpose is to preserve the approved visual quality across every screen size.

Desktop remains the reference.

Tablet and Mobile must adapt without losing hierarchy, rhythm, or storytelling.

---

# Philosophy

Responsive Design is not about making everything smaller.

Responsive Design is about preserving the experience.

Every visitor should experience the same story regardless of device.

# Review Checklist

## Hero

Verify:

- Typography remains dominant.
- CTA stays visible without scrolling.
- Hero image keeps its visual balance.
- Cards do not overlap unexpectedly.
- White space remains generous.
- First impression remains premium.

---

## Human Story

Verify:

- Image scales naturally.
- Story remains readable.
- Split layout collapses gracefully.
- Portrait remains emotionally impactful.
- Reading order stays logical.

---

## Portfolio

Verify:

- Cards stack naturally.
- Images keep correct aspect ratio.
- No cropped important subjects.
- Grid spacing remains consistent.
- Story rhythm is preserved.

---

## Full Width Sections

Verify:

- Images remain immersive.
- No excessive vertical height.
- Text overlays stay readable.
- Performance remains smooth.

---

## Impact Section

Verify:

- Statistics remain aligned.
- Numbers stay visually dominant.
- Cards do not wrap awkwardly.
- Typography hierarchy remains clear.

---

## Footer

Verify:

- Navigation remains usable.
- Contact information readable.
- CTA accessible.
- Comfortable spacing.

# Spacing Rules

Desktop

Generous

Tablet

Balanced

Avoid crowded layouts.

---

# Image Behaviour

Never crop faces.

Never crop important interactions.

Preserve storytelling over symmetry.

# Navigation

Horizontal

Mobile

# Interaction

Buttons

# Accessibility

Verify:

- Contrast
- Font size
- Touch targets
- Focus states
- Keyboard navigation
- Reduced motion compatibility

---

# Performance

Responsive implementation must not introduce:

- Layout Shift
- Image Flicker
- CLS
- Overflow
- Horizontal Scroll

---

---

# Definition of Done

Responsive implementation is complete when:

✓ Storytelling is preserved.

✓ Visual hierarchy remains consistent.

✓ Images remain meaningful.

✓ Typography remains premium.

✓ No layout breaks.

✓ Mobile feels intentionally designed.

✓ Desktop quality is maintained.

---

# Final Principle

Do not optimize for pixels.

Optimize for perception.

Every screen size should feel like it was designed intentionally.
