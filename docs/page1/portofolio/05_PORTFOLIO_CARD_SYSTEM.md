---

# Introduction

Portfolio Cards are not cards.

They are story containers.

The visitor should remember the project.

Not the UI component.

---

# Philosophy

Each card represents one transformation.

Never one service.

Never one product.

Never one image.

Every card contains a complete story.

---

# Card Hierarchy

Hero Card

↓

↓

↓

↓

Nothing more.

---

# Documentary Image

Largest element.

Always dominant.

Real photography.

Never illustration.

Never stock image.

---

# Category

Very small.

Uppercase.

Editorial.

Quiet.

Examples

COMMUNITY

MSME

TRAINING

BRANDING

INNOVATION

---

# Project Title

Short.

Strong.

Maximum two lines.

Emotion before information.

---

# Story

Maximum three lines.

Focus on transformation.

Never explain everything.

Invite curiosity.

---

# Metadata

Location

Year

Partner

Displayed quietly.

Never compete with the story.

---

# Evidence

Very small documentary annotation.

Examples

42 Participants

Village Program

Government Collaboration

Community Workshop

Displayed as evidence.

Not as marketing badges.

---

# CTA

Read Story

Explore Project

View Journey

Soft interaction.

Never "Learn More".

---

# Interaction

Hover

Very subtle lift.

Soft shadow.

Tiny image zoom.

Caption slightly brightens.

Nothing exaggerated.

---

# Motion

Fade.

Scale 1.02.

No rotation.

No bounce.

No dramatic movement.

---

# Checklist

✓ Story first.

✓ Documentary feeling.

✓ Human centered.

✓ Premium.

✓ Calm.

✓ Evidence driven.

---

# Final Principle

A Portfolio Card should feel like one page torn from a documentary journal.

Not like a UI component.

If visitors remember the people instead of the card, the design has succeeded.
