# Scope

Implement only the Navigation.

Do not modify Hero.

Do not modify About.

Do not modify Portfolio.

Do not modify Impact.

Do not modify Footer.

Preserve the existing architecture.

---

# Design Rules

Navigation must feel like one floating object.

Never become visually heavier than Hero.

Maintain the glass philosophy.

Maintain premium spacing.

Maintain documentary atmosphere.

Never introduce a SaaS navigation style.

---

# Visual Rules

Floating glass surface.

Soft backdrop blur.

Ambient lighting.

Large soft shadow.

Thin translucent border.

Rounded capsule.

Everything should feel refined.

---

# Motion Rules

Small opacity transitions.

Tiny translate.

Gentle hover.

Smooth scroll state.

No bounce.

No spring animations.

Respect prefers-reduced-motion.

---

# Engineering Rules

Strict TypeScript.

Reusable components.

Single responsibility.

Future CMS ready.

Future localization ready.

Production ready.

---

# Accessibility

Semantic nav.

Keyboard navigation.

Focus trap inside drawer.

Escape closes drawer.

Visible focus states.

WCAG AA compliant.

---

# Performance

GPU transforms only.

Passive scroll listeners.

Minimal JavaScript.

Avoid layout thrashing.

Maintain Lighthouse 95+.

---

# Final Review

Before returning code verify:

✓ Navigation feels lighter than Hero.

✓ Navigation remains readable.

✓ Glass feels premium.

✓ Blur remains subtle.

✓ Scroll behavior is smooth.

✓ Mobile drawer matches desktop language.

✓ Architecture is preserved.

✓ No unrelated files were modified.

---

# Final Principle

Navigation should feel like a silent guide.

Visitors should never admire the Navigation itself.

They should simply feel that moving through the website is effortless.
