# PROJECT GOAL

Build a premium editorial digital experience.

Not a SaaS website.

Not a startup landing page.

Not a corporate company profile.

The final experience should feel like a carefully crafted documentary where every section naturally leads to the next.

Visitors should remember the story,

never the interface.

---

# GLOBAL DESIGN RULES

Maintain one visual language.

Maintain one spacing rhythm.

Maintain one typography hierarchy.

Maintain one motion philosophy.

Maintain one lighting system.

Maintain one glass system.

Maintain one storytelling direction.

Never introduce a second design language.

---

# ATMOSPHERE

The interface should never feel flat.

Use atmosphere subtly.

Allowed:

• soft radial lighting

• ambient blur

• premium glass

• gentle reflections

• layered depth

• invisible 3D feeling

• elegant soft shadows

Never use:

Heavy glassmorphism.

Cyberpunk.

Gaming aesthetics.

Strong neon.

Aggressive gradients.

Visible visual effects.

Visitors should feel the atmosphere,

not notice it.

---

# ENGINEERING RULES

Do not rename folders.

Do not change architecture.

Do not duplicate logic.

Use reusable components.

Use strict TypeScript.

Keep the project CMS-ready.

Keep the project scalable.

Keep the project production-ready.

---

# PERFORMANCE

Prefer CSS over JavaScript whenever possible.

Optimize rendering.

Optimize images.

Minimize layout shifts.

Use GPU-friendly animations.

Maintain Lighthouse score above 95.

---

# ACCESSIBILITY

Semantic HTML.

Keyboard navigation.

Visible focus states.

Meaningful labels.

Reduced-motion support.

WCAG AA minimum.

Accessibility is mandatory.

---

# BEFORE RETURNING CODE

Always verify:

✓ Visual consistency.

✓ Storytelling consistency.

✓ Motion consistency.

✓ Lighting consistency.

✓ Glass consistency.

✓ Typography consistency.

✓ Responsive consistency.

✓ Accessibility.

✓ Performance.

✓ Clean architecture.

If any requirement is not satisfied,

continue refining before returning the implementation.

---

# OUTPUT RULES

Return only:

• Modified files

• Newly created files

• Short implementation notes

Never regenerate unrelated sections.

Never redesign completed work.

Only modify what is genuinely required.

---

# FINAL PRINCIPLE

Do not build pages.

Do not build components.

Do not build sections.

Build one seamless digital experience where every pixel belongs to the same story.

When visitors leave the UTBEX website,

they should remember the feeling,

the people,

and the impact—

never the technology behind it.
