# Introduction

Accessibility is not an additional feature.

It is a design requirement.

Every visitor deserves the same experience,

regardless of ability,

device,

or environment.

---

# Philosophy

Design for everyone.

Accessibility should never reduce elegance.

Elegance should never reduce accessibility.

The two must always coexist.

---

# Core Principles

Perceivable

↓

Operable

↓

Understandable

↓

Robust

Every component must satisfy these four principles.

---

# Readability

Typography must remain comfortable.

Maintain sufficient line height.

Maintain appropriate paragraph width.

Avoid low-contrast text.

Never sacrifice readability for aesthetics.

---

# Color Contrast

All text must satisfy WCAG AA.

Primary content should exceed minimum requirements whenever possible.

Never rely on color alone to communicate meaning.

---

# Keyboard Navigation

Every interactive element must be accessible using only the keyboard.

Users should always know where focus currently exists.

Focus should never disappear.

---

# Focus States

Focus indicators should feel premium.

Use:

• soft outline

• subtle glow

• thin accent border

Avoid flashing effects.

Avoid removing browser focus styles without replacement.

---

# Screen Reader Support

Every interactive element requires:

Meaningful labels.

ARIA attributes where necessary.

Logical reading order.

Avoid redundant announcements.

---

# Motion Accessibility

Respect

prefers-reduced-motion.

Disable:

Continuous ambient motion.

Parallax.

Floating animation.

Keep:

Opacity transitions.

Essential interaction feedback.

---

# Images

Decorative images

↓

Hidden from screen readers.

Content images

↓

Require meaningful alt text.

Never use generic descriptions.

---

# Forms

Every input requires:

Visible label.

Clear validation.

Helpful error messages.

Accessible focus order.

Never rely on placeholders as labels.

---

# Navigation

Navigation should always communicate:

Current location.

Available destinations.

Interaction state.

Mobile navigation must remain keyboard accessible.

---

# Responsive Accessibility

Touch targets

Minimum comfortable size.

Spacing prevents accidental interaction.

Typography remains readable.

Accessibility must remain consistent across all devices.

---

# Performance

Fast websites are more accessible.

Optimize:

Images.

Fonts.

JavaScript.

Animations.

Accessibility includes performance.

---

# Testing Checklist

✓ Keyboard only.

✓ Screen reader.

✓ Color contrast.

✓ Focus visibility.

✓ Responsive.

✓ Reduced motion.

✓ Semantic HTML.

✓ WCAG AA.

---

# Final Principle

The best accessibility is invisible.

Visitors should simply feel that the website works naturally for them,

without ever thinking about why.
