# UTBEX DIGITAL EXPERIENCE SYSTEM

## COMPONENT RULES

Version: 1.0

Status: LOCKED

Language: English

Audience:

Creative Director

UI Designer

Frontend Engineer

AI Coding Assistant

---

# Introduction

Every component belongs to one ecosystem.

No component should feel independently designed.

Every interaction,

every spacing,

every radius,

every shadow,

every animation,

must inherit the same visual language.

---

# Philosophy

Components should disappear behind the experience.

Visitors remember the story,

not the buttons,

cards,

or containers.

---

# Component Hierarchy

Navigation

↓

Hero

↓

Content Components

↓

Interactive Components

↓

Utility Components

Higher hierarchy receives more visual attention.

---

# Reusability

Every component should be:

Reusable.

Composable.

Predictable.

Maintainable.

CMS-ready.

Avoid creating one-time components.

---

# Visual Consistency

All components inherit:

• Color System

• Typography System

• Spacing System

• Surface System

• Lighting System

• Motion System

Never redefine these values locally.

---

# Component Anatomy

Every component consists of:

Container

↓

Content

↓

Interaction

↓

Feedback

↓

State

Maintain this order consistently.

---

# States

Every interactive component should support:

Default

↓

Hover

↓

Focus

↓

Active

↓

Disabled

↓

Loading (when applicable)

Never invent additional states unless necessary.

---

# Border Radius

Use one consistent radius scale.

Small

Medium

Large

Extra Large

Never mix unrelated corner styles.

---

# Shadow Usage

Only elevated components receive shadows.

Shadows remain soft.

Never stack multiple heavy shadows.

---

# Glass Components

Only use glass when hierarchy requires separation.

Examples:

Navigation

Floating Cards

Modal

Drawer

Never apply glass everywhere.

---

# Buttons

Primary

Highest priority.

Accent color.

Secondary

Neutral appearance.

Text Button

Lowest visual weight.

Maintain consistent sizing.

---

# Cards

Cards should never become decorative.

Their purpose is organization.

Use:

Comfortable padding.

Generous spacing.

Subtle elevation.

Clean typography.

---

# Forms

Forms should feel calm.

Simple labels.

Large inputs.

Comfortable spacing.

Clear validation.

No excessive decoration.

---

# Icons

Use one icon family.

Consistent stroke width.

Consistent visual weight.

Icons support text,

never replace it.

---

# Images

Images tell the story.

Components support images.

Never let decorative UI compete with photography.

---

# Interaction Feedback

Every interaction should receive feedback.

Opacity.

Shadow.

Underline.

Tiny movement.

Never dramatic animations.

---

# Accessibility

Every component must support:

Keyboard navigation.

Screen readers.

Visible focus.

Sufficient contrast.

WCAG AA minimum.

---

# Performance

Components should remain lightweight.

Avoid unnecessary wrappers.

Avoid duplicated logic.

Prefer composition over complexity.

---

# Component Checklist

✓ Reusable.

✓ Predictable.

✓ Accessible.

✓ Lightweight.

✓ Consistent.

✓ Premium.

✓ Editorial.

✓ Future-proof.

---

# Final Principle

A component is successful when it quietly supports the story without asking for attention.

Every component should feel like it naturally belongs to the UTBEX ecosystem.
